# Milestone 1.5 — Clip-Based Inference, CLI, ModelCar, Frame Export

## Overview

Extend the V-JEPA2 demo from a server-only container with baked-in model weights to a dual-mode container (server + CLI) with externalized models, clip-based inference as the unified pipeline, and frame export for debugging what the model sees.

**Milestone 1.5 scope**: Refactor inference around a clip iterator, add CLI subcommands (`serve`, `infer`, `download`), decouple model weights into ModelCar OCI images on Podman volumes, and export original/processed frames with contact sheets.

## Requirements

### Functional

- Unified entrypoint: `python -m app [serve|infer|download]`
- `serve`: starts FastAPI server (default subcommand, backward-compatible)
- `infer`: runs clip-based inference on video files, outputs predictions, optionally saves frames
- `download`: downloads a HuggingFace model into a volume mount
- Clip-based inference replaces uniform sampling as the primary pipeline for both file and stream paths
- Frame export: save original and preprocessed frames as PNGs, plus contact sheet grids
- Models loaded from a volume mount (`/model/`), not baked into the container image
- ModelCar OCI images for model distribution via `podman volume create --driver image`

### Non-Functional

- File-based and stream-based inference produce equivalent results for the same video
- CLI supports `--format text|json|jsonl` output formats
- ModelCar images use uncompressed layers for fast startup
- Multi-layer ModelCar images for parallel pull and cross-model deduplication
- Container image size reduced (~300MB without model, down from ~1.5GB)

### Out of Scope (Milestone 1.5)

- RTSP / webcam live stream ingestion (milestone 2)
- GPU-accelerated inference (milestone 2)
- KServe / Caikit integration (milestone 3)
- Kubernetes deployment (milestone 3, but ModelCar design is forward-compatible)

## Architecture

### Pipeline Refactor

The core change is unifying around a **clip iterator** as the single inference primitive. Both the API server and the CLI consume the same `iter_clips()` function, ensuring parity between file and stream delivery.

```text
video file ──► iter_clips(source, num_frames, stride) ──► Clip (T,H,W,3)
                  │                                            │
                  │ (yields one clip at a time)                │
                  │                                            ▼
                  │                                    model.predict(clip)
                  │                                            │
                  │                                            ▼
                  │                                    Prediction(label, score)
                  │
                  ├──► frames.save_clip_frames()  → original + processed PNGs
                  │         └──► contact sheets    → grid images
                  │
                  ├──► CLI: prints/streams results, saves frames to /output/
                  └──► API: returns JSON, streams via WebSocket
```

### Container Modes

```text
podman run vjepa2                              → defaults to serve
podman run vjepa2 serve                        → starts FastAPI on :8080
podman run vjepa2 infer                        → scans /input/, writes to /output/
podman run vjepa2 infer /input/hands.mp4       → specific file
podman run vjepa2 download --model <hf_id>     → downloads to /model/

Common volume mounts:
  -v model-vol:/model:ro       ← model weights (ModelCar volume)
  -v ./samples:/input:ro       ← input videos
  -v ./output:/output          ← frames, contact sheets, results
```

## Components

### app/video.py — Clip Iterator

The dual functions `decode_video()` and `stream_frames()` are replaced by a single `iter_clips()` iterator. `decode_video()` remains as a backward-compatible convenience wrapper.

```python
@dataclass
class Clip:
    frames: np.ndarray    # (num_frames, H, W, 3) uint8
    start_frame: int      # index in source video
    end_frame: int        # exclusive end index

def iter_clips(
    source: str | Path,
    num_frames: int = 16,
    stride: int | None = None,   # None → non-overlapping (stride=num_frames)
) -> Iterator[Clip]:
    """Yield clips from a video source.

    Decodes frames sequentially and yields clips as sliding windows.
    When stride < num_frames, clips overlap. When stride == num_frames,
    clips are non-overlapping.

    Trailing frames (fewer than num_frames remaining) are padded by
    repeating the last frame. The Clip's end_frame reflects the actual
    video frames (not padding), so callers can detect partial clips.
    """

def decode_video(source: str | Path, num_frames: int) -> np.ndarray:
    """Convenience: return frames from first clip.
    Backward-compatible with milestone 1 callers."""
    clip = next(iter_clips(source, num_frames, stride=num_frames))
    return clip.frames
```

**Trailing frames**: When the video length doesn't divide evenly by stride, the final clip is padded by repeating the last frame. The clip is marked as partial via `end_frame < start_frame + num_frames`. This ensures full video coverage — no frames are silently dropped.

**Parity guarantee**: Both the REST API and CLI use `iter_clips()`. A video processed locally produces the same clip boundaries and predictions as the same video streamed via WebSocket. The only difference is delivery mechanism.

### app/cli.py — CLI Subcommands

```python
def cmd_serve(args):
    """Start FastAPI server via uvicorn."""

def cmd_infer(args):
    """Run clip-based inference on video files.

    If no files specified, scans /input/ for video files.
    Outputs predictions to stdout in the requested format.
    Optionally saves frames to --output directory.

    Args:
        files: video file paths (optional, default: scan /input/)
        --stride: clip stride (default: num_frames = non-overlapping)
        --num-frames: frames per clip (default: 16)
        --top-k: predictions per clip (default: 5)
        --save-frames: save original + processed PNGs and contact sheets
        --output: output directory (default: /output)
        --format: output format — text, json, jsonl (default: text)
    """

def cmd_download(args):
    """Download a HuggingFace model into a directory.

    Args:
        --model: HuggingFace model ID (required)
        --output: target directory (default: /model)
    """
```

#### Output Formats

**`--format text`** (default): Human-readable output with progress to stderr.

```text
Clip 0 [frames 0–15]
  0.872  Pushing something from left to right
  0.061  Moving something across a surface
```

**`--format jsonl`**: One JSON object per clip, streamed to stdout as each clip completes. Progress/status to stderr.

```jsonl
{"clip_index":0,"start_frame":0,"end_frame":16,"partial":false,"predictions":[{"label":"Pushing something from left to right","score":0.872}]}
{"clip_index":1,"start_frame":16,"end_frame":32,"partial":false,"predictions":[{"label":"Picking something up","score":0.743}]}
```

**`--format json`**: Complete JSON array written after all clips are processed.

```json
[
  {"clip_index":0,"start_frame":0,"end_frame":16,"partial":false,"predictions":[...]},
  {"clip_index":1,"start_frame":16,"end_frame":32,"partial":false,"predictions":[...]}
]
```

### Containerfile

```dockerfile
FROM python:3.12-slim

WORKDIR /app

# System dependencies for PyAV
RUN apt-get update && \
    apt-get install -y --no-install-recommends \
      libavcodec-dev libavformat-dev libavutil-dev libswscale-dev && \
    rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir \
      --index-url https://download.pytorch.org/whl/cpu torch torchvision && \
    pip install --no-cache-dir -r requirements.txt

COPY app/ ./app/
COPY configs/ ./configs/

# No model weights — loaded from volume at runtime
VOLUME /model
VOLUME /input
VOLUME /output

RUN mkdir -p /input /output

EXPOSE 8080

ENTRYPOINT ["python", "-m", "app"]
CMD ["serve"]
```

No downloader stage. Model weights come from a volume mount.

### Modelfile (ModelCar OCI Image)

Located at the repo root as `Modelfile`.

```dockerfile
FROM scratch

# Layer 1: config + preprocessor (small, shared across variants)
COPY model/config.json              /model/
COPY model/preprocessor_config.json /model/

# Layer 2+: weight shards (separate layers for parallel pull)
COPY model/model.safetensors        /model/
```

Build and distribute:

```bash
# Download model to local staging directory
podman run -v ./model-staging:/model vjepa2 \
  download --model facebook/vjepa2-vitl-fpc16-256-ssv2

# Build ModelCar image (uncompressed for fast startup)
podman build \
  --disable-compression \
  -f Modelfile \
  -t quay.io/fzdarsky/vjepa2-vitl:latest \
  .

# Push to registry
podman push quay.io/fzdarsky/vjepa2-vitl:latest
```

Consume:

```bash
# Create volume from ModelCar (single step)
podman volume create \
  --driver image \
  --opt image=quay.io/fzdarsky/vjepa2-vitl:latest \
  vjepa2-vitl

# Serve
podman run -v vjepa2-vitl:/model:ro -p 8080:8080 vjepa2 serve
```

Kubernetes (future):

```yaml
volumes:
  - name: model
    image:
      reference: quay.io/fzdarsky/vjepa2-vitl:latest
      pullPolicy: IfNotPresent
```

**Layer optimization**: Each `COPY` in the Modelfile creates a separate OCI layer. Uncompressed layers (`--disable-compression`) skip decompression of dense float tensors at volume creation time. For multi-shard models, each shard gets its own layer for parallel registry pull. Fine-tuned variants sharing the same config layer get deduplication automatically.

### API Changes

| Endpoint | Change |
| -------- | ------ |
| `POST /v2/models/vjepa2/infer` | Add optional `stride` form param. When set, returns per-clip predictions. Default behavior unchanged. |
| `WS /v2/models/vjepa2/stream` | Internal refactor to use `iter_clips()`. The WebSocket already receives a file path via `StreamConfig.source` — `iter_clips()` replaces the internal `stream_frames()` call. Client-facing protocol unchanged. |
| `GET /v2/models/vjepa2` | Model loaded from `/model/` path. Version derived from model config. |

## Error Handling

| Scenario | Behavior |
| -------- | -------- |
| No model at `/model/` | `serve`/`infer` fail fast with clear error: "No model found at /model/. Run `download` first or mount a model volume." |
| No videos in `/input/` | `infer` exits with message: "No video files found in /input/" |
| Video shorter than num_frames | Single partial clip (padded), marked in output |
| Partial trailing clip | Padded by repeating last frame, `partial: true` in manifest |
| `--save-frames` without writable `/output/` | Error: "Output directory /output is not writable" |
